项目已经内置了演示用的老师账号、组长账号、成员账号、团队任务、公告、评分和团队文件。

只要按下面的步骤依次操作，就可以启动项目。

## 拉下代码后先做什么

这是最重要的部分。  
如果你什么都不想看，只想先跑起来，就照着下面 6 步做。

### 1. 安装 Node.js

先确认电脑里已经安装了 Node.js。

推荐版本：

- Node.js 20 或更高版本

检查方法：

```bash
node -v
```

如果能看到版本号，例如 `v20.x.x`，说明已经安装好了。

如果没有安装，请先去 Node.js 官网下载安装：

- https://nodejs.org/

### 2. 安装项目依赖

进入项目目录后，执行：

```bash
npm install
```

如果你习惯用 `pnpm`，也可以执行：

```bash
pnpm install
```

说明：

- 这一步是在下载项目运行需要的依赖包
- 第一次执行可能需要几分钟，属于正常现象

### 3. 创建 `.env` 文件

本项目需要一个本地环境变量文件。

最简单的做法是把 `.env.example` 复制成 `.env`。

Windows PowerShell：

```powershell
Copy-Item .env.example .env
```

Mac / Linux：

```bash
cp .env.example .env
```

如果你不想复制，也可以手动新建一个名为 `.env` 的文件，内容如下：

```env
DATABASE_URL="file:./dev.db"
```

### 4. 初始化数据库和演示数据

执行下面这条命令：

```bash
pnpm run db:setup
```

说明：

- 这一步会自动创建本地数据库
- 会自动写入演示账号
- 会自动写入团队、任务、公告、评分、团队文件等展示数据

如果你以后想把演示数据重置回初始状态，也可以再次执行这条命令。

### 5. 启动项目

开发模式启动：

```bash
pnpm dev
```

启动成功后，在浏览器打开：

- http://localhost:3000

### 6. 如果要做正式展示，推荐这样启动

如果你想用“更接近交付展示”的方式运行，而不是开发模式，建议使用下面两条命令：

```bash
pnpm build
pnpm start
```

然后在浏览器打开：

- http://localhost:3000

## 常见问题

### 1. 拉下来代码后能直接登录吗？

不能直接登录。

不是因为要重新注册账号，而是因为本地数据库和演示数据不会上传到 GitHub。

所以必须先执行：

```bash
pnpm run db:setup
```

执行完成后，演示账号才会自动生成。

### 2. 为什么项目里没有现成的数据库文件？

因为数据库文件 `dev.db` 是本地文件，不应该提交到仓库。

每个人在自己的电脑上运行 `pnpm run db:setup` 后，会自动生成自己的本地数据库。

### 3. 为什么项目里要有 `.env` 文件？

因为项目需要知道数据库文件放在哪里。

本项目目前只需要这一项配置：

```env
DATABASE_URL="file:./dev.db"
```

### 4. 启动后页面打不开怎么办？

按顺序检查：

1. 有没有执行 `pnpm install`
2. 有没有创建 `.env`
3. 有没有执行 `pnpm run db:setup`
4. 是不是已经执行了 `pnpm dev` 或 `pnpm start`
5. 浏览器打开的是不是 `http://localhost:3000`

### 5. 如果我想重置演示数据怎么办？

直接执行：

```bash
pnpm run db:setup
```

或者：

```bash
pnpm run db:seed
```

这样会把演示数据恢复成项目预设状态。
